brs_92 media stop 시 이미지를 외부로 어떻게 내려보내? (ccnc)
실질적 이미지 교체는 STOPPED가 아닌 LOADING 전환 시 발생한다.
전체 호출 체인
updateAudioState(kClosed)
  → onPlayingStateChanged(kClosed)
      → removeMediaSessionInfo()
      → changePlayingState(kClosed)
          → resetLastMediaInfo()
          → updateMediaState(MediaWidgetStatus::LOADING, app_id, -1)
              → nowplaying_controller_.sendDefaultNowPlayingInfo()
                  → sendImageUrl(app_id, getDefaultImagePath(app_id))
                      → port::Audio::setImageUrl(app_id, source, defaultImage, LOADING)
                          → setNowPlayingImage(screen, app_id, bgImg=logoImg, albumImg=logoImg)
상태별 동작 비교
상태 port::Audio::setImageUrl 실제 이미지 변화
LOADING (kClosed 경유) 호출됨 — 앱 로고(기본 이미지) 전달 썸네일 → 앱 로고로 교체
STOPPED 미호출 — display_control_.updateMediaStatus(STOPPED, "") 만 이미지 유지, 위젯 상태만 변경
핵심 코드 (brs_92 app_policy.cc)
case MediaWidgetStatus::STOPPED: {
    // setImageUrl 호출 없음, 위젯 상태만 변경
    display_control_.updateMediaStatus(source, STOPPED, app_id, "");
} break;

case MediaWidgetStatus::LOADING: {
    // 기본 이미지(앱 로고)로 교체
    nowplaying_controller_.sendDefaultNowPlayingInfo(app_id, force, status);
    // → port::Audio::setImageUrl(app_id, source, logoImage, LOADING)
    display_control_.updateMediaStatus(source, LOADING, app_id, defaultImagePath);
} break;
brs_143 에 대한 시사점
현재 brs_143에 적용한 수정 (onMediaSessionPlaybackStatusChanged에서 availability=false 시 updateMediaState(STOPPED))은 brs_92 방식 기준으로 이미지를 건드리지 않는 방향이다.

실제 이미지 제거/교체는 LOADING 상태를 통해 기본 이미지를 보내는 방식이므로, 썸네일을 실제로 지우려면 아래처럼 변경하는 것이 brs_92와 동일한 동작이다. 
if (!availability) {
    updateMediaState(MediaWidgetStatus::LOADING, app_id_, -1);  // STOPPED → LOADING
}
Obigo 추가 구현 요약 — MediaSessionController → 외부 image/playing info 연결
원본(~/server/143origin) 대비 2가지 계층으로 구현되어 있다.
1. 신규 파일 — content/browser/media/obigo/ (원본에 없음)
obigo_media_session_handler.h/.cc — 핵심 외부 연결 컴포넌트. 원본 Chromium에는 존재하지 않는 obigo 전용 클래스.
구현 인터페이스역할
MediaControllerObserver play state / metadata / actions 변화 수신
MediaControllerImageObserver artwork 이미지 변화 수신
생성자에서 MediaControllerManager로부터 active controller를 생성하고 두 observer를 등록: 
// ObserveImages: kArtwork, minSize=71, desiredSize=300
media_controller_->ObserveImages(kArtwork, 71, 300, image_observer_receiver_);
각 이벤트를 ShellBrowserWSClient를 통해 외부로 전달:
콜백외부 전달 함수
MediaSessionInfoChanged() wsClient->NotifyPlayStateChanged(media_id, playState, availability)
MediaSessionMetadataChanged() wsClient->NotifyMetaDataChanged(media_id, title, artist)
MediaSessionActionsChanged() wsClient->NotifyActionsChanged(actionList)
MediaControllerImageChanged() wsClient->NotifyImageChanged(hasThumbnail)
2. 기존 파일 수정 — #if defined(OBIGO_AUDIO_MANAGER)
