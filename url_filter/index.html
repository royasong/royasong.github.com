JSON 입력 포맷 — Map vs List

Map (Dict)
도메인을 key로, 경로 패턴 목록을 value로 지정합니다.
경로 패턴의 첫 글자로 타입이 결정됩니다 (/ → PathPrefix, = → ExactPath, * → Wildcard).
빈 배열([])이면 루트(/) 접근만 차단합니다.
{ "naver.com": ["/news", "=/pay", "*tracker"], "kakao.com": [] // 루트만 차단 }
List (Array)
URL 전체를 나열합니다. 파싱 시 host를 key로 추출하고,
값은 "@" + URL spec 형태로 저장되어 kUrlPrefix 타입으로 컴파일됩니다.
[ "https://naver.com/news", "https://m.naver.com/audio-car" ] → result["naver.com"] = ["@https://naver.com/news"] → result["m.naver.com"] = ["@https://m.naver.com/audio-car"]
이중 인코딩
값이 JSON 문자열로 한 번 더 직렬화된 경우 자동으로 재파싱합니다.
재파싱 결과가 List이면 위 List 방식, Dict이면 Map 방식으로 처리합니다.
{ "whiteList": "[\"https://naver.com/\"]" } ↑ 값 자체가 string → JSONReader::Read() 재파싱

"@" 접두사의 의미

생성 시점
List 포맷에서 URL을 파싱할 때 ParseDomainPatternMap이 자동으로 붙입니다.
직접 Map 포맷에서 사용할 수도 있습니다.
result[host].push_back("@" + gurl.spec()); // "https://naver.com/news" → "@https://naver.com/news"
컴파일 결과
CompilePattern()에서 @를 감지하면 kUrlPrefix 타입으로 컴파일합니다.
@를 제거한 나머지 URL 문자열이 value로 저장됩니다.
raw = "@https://naver.com/news" → type = kUrlPrefix → value = "https://naver.com/news" // "@" 제거 → subdomain_host = "" // 하위 경로 → fallback 없음 raw = "@https://naver.com/" // 루트 URL → type = kUrlPrefix → value = "https://naver.com/" → subdomain_host = "naver.com" // 서브도메인 fallback 세팅
매칭 방식
전체 URL spec이 value로 시작(prefix)하면 매칭합니다.
루트 URL인 경우 subdomain_host로 DomainIs()를 추가 체크해 서브도메인도 매칭합니다.
value = "https://naver.com/news" "https://naver.com/news" → MATCHED (정확히 일치) "https://naver.com/news/article" → MATCHED (prefix 시작) "https://naver.com/" → NO MATCH (prefix 불일치) "https://m.naver.com/news" → NO MATCH (subdomain_host 없음) value = "https://naver.com/" (루트, subdomain_host="naver.com") "https://naver.com/any/path" → MATCHED (prefix 시작) "https://m.naver.com/page" → MATCHED (DomainIs fallback)
