URL Filter Domain Rule 설계 문서

파일: src/components/obigo/url_filter/core/url_filter_rules.cc

1. 개요

UrlFilter는 브라우저가 HTTP/HTTPS URL을 탐색하려 할 때, 해당 URL이 허용(ALLOWED), 차단(BLOCKED), 미지원(NOT_SUPPORTED) 중 어느 상태인지를 판별하는 컴포넌트입니다.

판별에 사용되는 멤버 변수는 다음과 같습니다:

필드/맵역할매칭 시 결과
host_url_앱 자체 호스트 URL (문자열)
Init() 첫 번째 인자로 설정		kUrlAllowed ※ 최우선 체크
unsupported_domain_map_지원되지 않는 도메인/경로kUrlNotSupported
black_domain_map_차단 도메인/경로kUrlBlocked
white_domain_map_허용 도메인/경로kUrlAllowed
host_url_ 최우선 체크:
Init()에서 설정된 host_url_이 비어 있지 않고, 탐색 URL이 host_url_과 정확히 일치하면 unsupported / blacklist / whitelist 체크 없이 즉시 kUrlAllowed를 반환합니다.
앱 자체의 시작 URL(홈 화면 등)이 필터 규칙에 의해 차단되는 상황을 방지합니다.

판별 우선순위

① host_url_ 일치 → 즉시 ALLOWED (정확 일치)
② 비HTTP/HTTPS 스킴 → 즉시 ALLOWED (about:, file:, blob: 등)
③ kAlwaysAllowed 패턴 → 즉시 ALLOWED (chrome-distiller 등)
④ unsupported_domain_map_ → NOT_SUPPORTED (whitelist보다 먼저)
⑤ kAlwaysBlocked 패턴 → 즉시 BLOCKED
⑥ black_domain_map_ → BLOCKED
⑦ white_domain_map_ → ALLOWED / BLOCKED (miss 시 BLOCKED)

3. CompiledPattern 타입 (5종)

raw 패턴 문자열의 첫 글자로 타입이 결정됩니다:

타입raw 접두사매칭 대상예시
kUrlPrefix @ 전체 URL spec이 value로 시작
+ 루트일 때 서브도메인 DomainIs 체크		 @https://naver.com/news
kExactPath = path가 value와 정확히 일치
(=/ 는 루트 포함)		 =/about
kWildcard * path 어딘가에 value 포함
(value 비어있으면 전체 매칭)		 *sports
kPathPrefix / path가 value로 시작 /news
kBareSubstring (없음) 전체 URL spec 어딘가에 포함 tracker
kUrlPrefix의 서브도메인 fallback:
raw 패턴이 @https://naver.com/(루트 URL)인 경우, subdomain_host="naver.com"이 세팅됩니다.
이후 m.naver.com, news.naver.com 등 서브도메인도 DomainIs("naver.com")으로 함께 매칭됩니다.

4. JSON 입력 포맷 (ParseDomainPatternMap)

세 가지 JSON 구조를 모두 지원합니다:

① Dict 형식 — 도메인별 경로 목록

{
  "naver.com": ["/news", "/sports"],   // 특정 경로만 적용
  "kakao.com": [],                      // 빈 배열([]) = 루트("/")만 차단
  "open.spotify.com": [""]             // 빈 문자열([""])= 전체 경로 차단
}

raw 패턴에 접두사 없이 저장 → kPathPrefix 또는 kBareSubstring으로 컴파일.

[] vs [""] 차이:
[] (빈 배열) — rule.second.empty() 분기로 처리. path가 /이거나 없는 경우(루트)만 차단.
[""] (빈 문자열 원소) — CompilePattern("") → kBareSubstring 빈값. spec.find("") == 0은 항상 성립하므로 모든 경로 차단.

② List 형식 — URL 전체 목록

["https://naver.com/news", "https://kakao.com/"]

각 URL의 host를 key로, "@" + spec을 value로 저장.
→ result["naver.com"] = ["@https://naver.com/news"] → kUrlPrefix로 컴파일.

③ 이중 인코딩된 JSON 문자열

"whiteList": "[\"https://naver.com/news\"]"   // 값이 string

root.FindString(key)로 감지 후 base::JSONReader::Read()로 재파싱.
재파싱 결과가 List이면 ② 방식, Dict이면 ① 방식으로 처리.

5. Rule 빌드 파이프라인

JSON 설정 │ └─ ParseDomainPatternMap() 결과: DomainPatternMap = map<string, vector<string>> = { "naver.com" → ["/news", "/sports"] } │ └─ BuildCompiledMap() ① NormalizeDomainPathRule() : host 소문자 정규화 host에 path가 있으면 "=path" 패턴 prepend ② CompilePattern(raw) : 각 raw 문자열 → CompiledPattern{type, value, subdomain_host} 결과: CompiledPatternMap = map<string, CompiledPatternList> = { "naver.com" → [{kPathPrefix, "/news"}, {kPathPrefix, "/sports"}] } │ └─ unsupported_domain_map_ / black_domain_map_ / white_domain_map_ 에 저장

6. NormalizeDomainPathRule 동작

Dict 형식에서 key(host)에 URL 전체가 들어올 수 있습니다. 이를 정규화합니다:

입력 host처리결과 hostpaths에 추가
naver.com https:// 붙여 파싱 naver.com 변경 없음
https://naver.com/news URL 파싱 후 host 추출 naver.com =/news 앞에 삽입
NAVER.COM 소문자 변환 naver.com 변경 없음

7. DomainIs 매칭 규칙

gurl.DomainIs("example.com")은 host의 뒷부분 일치 여부를 확인합니다.
단순 suffix 비교가 아니라, host가 더 길 경우 바로 앞에 .이 있어야 합니다.

URL hostdomain결과이유
example.comexample.com✅ true완전 일치
www.example.comexample.com✅ true앞에 . 있음
sub.example.comexample.com✅ true앞에 . 있음
notexample.comexample.com❌ false앞에 . 없음
other.comexample.com❌ falsesuffix 불일치

8. kUrlNotSupported vs kUrlBlocked 차이

반환값의미UI 표현 (예시)
kUrlNotSupported 브라우저가 지원하지 않는 사이트
(기능상 제한, 정책상 비활성)		 "지원되지 않는 사이트입니다" 안내 페이지
kUrlBlocked 명시적으로 차단된 URL
(보안, 콘텐츠 정책)		 "차단된 페이지입니다" 경고 페이지

unsupported 체크는 blacklist/whitelist 체크보다 먼저 실행됩니다. 즉, unsupported 도메인은 whitelist에 있어도 kUrlNotSupported가 반환됩니다.

9. 동작 예시

시나리오 1: unsupported 루트 차단

// JSON 설정
{ "unsupported": { "naver.com": [] } }

// 판별 결과
"https://naver.com/"          → NOT_SUPPORTED  (루트 매칭)
"https://m.naver.com/"        → NOT_SUPPORTED  (서브도메인, DomainIs)
"https://naver.com/news"      → 이후 black/white 로 진행 (루트 아님)

시나리오 2: 특정 경로만 차단

// JSON 설정
{ "blackList": { "naver.com": ["/sports"] } }

// 판별 결과
"https://naver.com/sports"         → BLOCKED  (kPathPrefix 매칭)
"https://naver.com/sports/soccer"  → BLOCKED  (prefix 포함)
"https://naver.com/news"           → ALLOWED  (blacklist miss → whitelist 없음)

시나리오 3: List 형식 whitelist

// JSON 설정
{ "whiteList": ["https://naver.com/"] }

// 파싱 결과
result["naver.com"] = ["@https://naver.com/"]
CompilePattern("@https://naver.com/")
  → { kUrlPrefix, "https://naver.com/", subdomain_host="naver.com" }

// 판별 결과
"https://naver.com/news"    → ALLOWED  (kUrlPrefix prefix 매칭)
"https://m.naver.com/page"  → ALLOWED  (DomainIs subdomain fallback)
"https://daum.net/"         → BLOCKED  (whitelist miss)

시나리오 4: 빈 문자열 패턴으로 사이트 전체 차단

// JSON 설정
{ "blackList": { "open.spotify.com": [""] } }

// 컴파일
CompilePattern("") → { kBareSubstring, value="", subdomain_host="" }
spec.find("") == 0  → 항상 true → 모든 URL에 매칭

// 판별 결과
"https://open.spotify.com/"          → BLOCKED  (kBareSubstring 빈값 → 항상 매칭)
"https://open.spotify.com/track/123" → BLOCKED  (kBareSubstring 빈값 → 항상 매칭)
"https://sub.open.spotify.com/"      → BLOCKED  (DomainIs 일치 + kBareSubstring 항상 매칭)
"https://spotify.com/"               → ALLOWED  (DomainIs 불일치 — open.spotify.com ≠ spotify.com)

// [] vs [""] 비교 요약
"kakao.com": []   → 루트("/") 만 차단,  "/news" 등 하위 경로는 통과
"kakao.com": [""] → 전체 경로 차단  (kBareSubstring 빈값이 spec 어디에나 매칭)

시나리오 5: host_url_ 최우선 허용

// Init() 설정
host_url = "https://home.myapp.com/"
blackList = { "myapp.com": [] }   // myapp.com 전체 차단 규칙

// 판별 결과
"https://home.myapp.com/"    → ALLOWED  (host_url 정확 일치 → blacklist 체크 전 즉시 반환)
"https://other.myapp.com/"   → BLOCKED  (host_url 불일치 → blacklist 루트 매칭)
"https://home.myapp.com/sub" → BLOCKED  (host_url 불일치 — 정확 일치만 해당 → blacklist 매칭)

10. Path Pattern 동작 정리

경로 패턴 문자열의 형태에 따라 컴파일 타입과 매칭 동작이 달라집니다. 패턴을 잘못 선택하면 의도치 않은 URL이 차단되거나 누락될 수 있습니다.

unittest: src/components/obigo/url_filter/core/url_filter_rules_unittest.cc

# 패턴 (raw) 컴파일 타입 매칭 대상 조건 차단 예시 통과 예시 비고 관련 테스트
1 */duo/ kWildcard gurl.path() path.find("/duo/") != npos /duo/
/kr-ko/duo/		 /duo-show/
?q=duo		 path 내 /duo/가 포함된 모든 경우 Pattern1_WildcardSubpathContains
2 * (단독) kWildcard
value=""		 — 항상 true 모든 URL — value가 비어있어 전체 URL 차단. [""]의 kBareSubstring 빈값과 동일 효과 Pattern2_BareWildcardMatchesAll
3 /duo/ kPathPrefix gurl.path() path.find("/duo/") == 0 /duo/
/duo/anything		 /kr-ko/duo/ path가 정확히 /duo/로 시작해야 매칭.
후행 슬래시 없는 /duo도 불일치		 Pattern3_AbsolutePrefixStartsWith
4 duo/ kBareSubstring gurl.spec() spec.find("duo/") != npos /duo/
/kr-ko/duo/		 — ⚠️ 오탐 주의. abcduo/처럼 접두어가 붙은 문자열도 매칭됨 Pattern4_BareSubstringSpecWide
5 "host": []
"host/path": []		 — (빈 배열)
kExactPath		 gurl.path() path == / 또는 비어있음
(key에 path 있으면 =path 자동 삽입)		 루트 /
또는 지정 path		 모든 하위 경로 정확히 일치하는 URL만 차단.
NormalizeDomainPathRule이 key의 path를 =path 패턴으로 변환		 Pattern5_HostOnly_EmptyPaths_RootOnly
Pattern5_HostPathKeyNormalized
6 =/ kExactPath
value="/"		 gurl.path() path == / 또는 비어있음 루트 / 만 모든 하위 경로 루트만 명시적으로 차단. []과 동일 효과이나 다른 패턴과 배열로 혼합 가능 Pattern6_RootAndDuoUnsupportedCombined

혼합 패턴 예시 — 루트 + 특정 경로 동시 차단

루트(홈)와 특정 경로를 함께 차단해야 하는 경우 =/와 * 패턴을 배열로 조합합니다.
key에 scheme(http://)이 있어도 NormalizeDomainPathRule이 host만 추출하므로 http/https 모두 매칭됩니다.
→ unittest: Pattern6_RootAndDuoUnsupportedCombined / SpotifyFilterTest (www.spotify.com blackList) 

// JSON 설정 (unsupported 예시 — blackList도 동일하게 동작)
{
  "unsupportedList": {
    "http://www.spotify.com": [   // scheme 포함 key → NormalizeDomainPathRule이 "www.spotify.com" 추출
      "*/duo/",                    // path 내 /duo/ 포함 → NOT_SUPPORTED
      "=/"                         // 루트(/) 정확 일치 → NOT_SUPPORTED
    ]
  }
}

// 판별 결과
"https://www.spotify.com/"           → NOT_SUPPORTED  (=/ 루트 매칭)
"https://www.spotify.com/duo/"       → NOT_SUPPORTED  (*/duo/ 매칭)
"https://www.spotify.com/kr-ko/duo/" → NOT_SUPPORTED  (*/duo/ 매칭)
"https://www.spotify.com/duo-show/"  → ALLOWED        (*/duo/ 불일치 — /duo-show/에는 /duo/가 없음)
"https://www.spotify.com/premium/"   → ALLOWED        (두 패턴 모두 불일치)
"https://spotify.com/"               → ALLOWED        (DomainIs 불일치 — www.spotify.com ≠ spotify.com)
패턴 선택 가이드:
• 사이트 전체 차단: [] (루트만) / [""] 또는 ["*"] (전체)
• 특정 경로로 시작: /path/ → kPathPrefix (정확한 시작점)
• 경로 어딘가 포함: */keyword/ → kWildcard (슬래시로 감싸 오탐 방지)
• 루트 + 특정 경로 혼합: ["=/", "*/keyword/"] 배열 조합
