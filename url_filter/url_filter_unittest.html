1. UrlFilterTest — 핵심 로직 단위 테스트

가장 기본적인 테스트 클래스입니다. 설정을 직접 C++ Map으로 구성해 각 동작을 독립적으로 검증합니다.
주요 그룹: Whitelist 동작 · Blacklist 동작 · Unsupported 동작 · 패턴 타입 5종 · 처리 우선순위 · 재초기화
// 각 TEST_F마다 SetUp에서 필요한 맵을 직접 구성 // 예) WhitelistPrefixPath DomainPatternMap white = {{"naver.com", {"/news"}}}; filter_.Init("", white, {}); // → /news, /news/article → ALLOWED / /sports → BLOCKED

테스트명검증 내용결과
WhitelistEmptyAllowsAll black/white 모두 비어있으면 모든 URL 허용 ALLOWED
WhitelistDomainOnly 패턴 없는 whitelist → 도메인 전체 경로 허용 (patterns.empty()==true) ALLOWED

2. SpotifyFilterTest — Spotify 실서비스 설정 (C++ Map)

Spotify 앱에 적용된 실제 멀티도메인 설정입니다. whitelist는 C++ Map Key에 URL을 넣어 NormalizeDomainPathRule로 host+path 분리, blacklist는 도메인+경로 패턴 방식을 사용합니다.
// Whitelist — Map Key에 URL 전체를 넣으면 NormalizeDomainPathRule이 자동 분리 DomainPatternMap white = { {"https://open.spotify.com/", {}}, // host=open.spotify.com, path=/ → 패턴 없음 → 전체 허용 {"https://appleid.apple.com/auth/", {}}, // host=appleid.apple.com, "=/auth/" 패턴 삽입 → /auth/ 만 허용 {"https://www.fbsbx.com/captcha", {}}, // "=/captcha" 패턴 삽입 {"https://accounts.spotify.com/", {}}, {"https://accounts.spotify.com", {}}, // 동일 host 중복 → merge됨 ... }; // Blacklist — 도메인 + 경로 패턴 DomainPatternMap black = { {"open.spotify.com", {"/download/", "*/premium/", "?go"}}, {"www.spotify.com", {"*/duo/", "=/"", ...}}, {"facebook.com", {"/Spotify/", "/watch/"}}, }; // Unsupported — facebook OAuth 팝업 DomainPatternMap unsupported = { {"facebook.com", {"dialog/oauth"}}, // kBareSubstring {"m.facebook.com", {}}, // 루트만 차단 };
Host URL결과판단 근거
https://open.spotify.com/ ALLOWED Blacklist miss (루트) → Whitelist 패턴 없음(전체 허용)
https://open.spotify.com/track/abc ALLOWED Blacklist miss → Whitelist 전체 허용
https://open.spotify.com/download/win BLOCKED Blacklist kPathPrefix /download/ 매칭
https://open.spotify.com/kr/premium/ BLOCKED Blacklist kWildcard */premium/: path에 "premium/" 포함
https://appleid.apple.com/auth/ ALLOWED Whitelist kExactPath =/auth/ 매칭
https://appleid.apple.com/login BLOCKED Whitelist =/auth/ 불일치 → miss → BLOCKED
https://www.fbsbx.com/captcha ALLOWED Whitelist kExactPath =/captcha 매칭
https://www.fbsbx.com/other BLOCKED Whitelist =/captcha 불일치 → BLOCKED
https://facebook.com/dialog/oauth/cb NOT_SUPPORTED Unsupported kBareSubstring dialog/oauth spec 포함
https://m.facebook.com/ NOT_SUPPORTED Unsupported 패턴 없음 + 루트 → NOT_SUPPORTED
https://www.spotify.com/ BLOCKED Blacklist kExactPath =/ 루트 정확 매칭
https://payments.spotify.com/ BLOCKED Blacklist 패턴 없음 + 루트 → BLOCKED

3. SpotifyConfigTest — Spotify JSON 파싱 정합성 검증

SpotifyFilterTest와 동일한 설정을 JSON 문자열로 파싱한 뒤 같은 결과가 나오는지 확인합니다.
ParseDomainPatternMap()의 파싱이 C++ Map 직접 구성과 동일하게 동작함을 보장합니다.
// JSON 포맷으로 동일 설정 표현 constexpr char kJson[] = R"({ "whiteList": [ "https://open.spotify.com/", "https://appleid.apple.com/auth/", "https://www.fbsbx.com/captcha", ... ], "blackList": { "open.spotify.com": ["/download/", "*/premium/", "?go"], "www.spotify.com": ["*/duo/", "=/", ...], ... }, "unSupportedDomainList": { "facebook.com": ["dialog/oauth"], "m.facebook.com": [] } })"; // → ParseDomainPatternMap으로 파싱 → SpotifyFilterTest와 동일한 결과 기대
SpotifyFilterTest와 동일한 URL 케이스를 반복 검증하여 JSON 파싱 → 필터 동작 경로가 C++ 직접 구성과 일치함을 확인합니다. 결과가 다르면 ParseDomainPatternMap 또는 NormalizeDomainPathRule에 버그가 있다는 신호입니다.

4. ParseDomainPatternMapTest — JSON 파싱 전용 테스트

ParseDomainPatternMap() 함수 자체만 집중 검증합니다. 필터 동작이 아니라 파싱 결과(DomainPatternMap)가 올바른지 확인합니다.
// 대표 케이스들 // ① Array 포맷 — URL 목록 "whiteList": ["https://naver.com/news", "https://m.naver.com/"] // → result["naver.com"] = ["@https://naver.com/news"] // → result["m.naver.com"] = ["@https://m.naver.com/"] (루트 → subdomain_host 세팅) // ② Dict 포맷 — 도메인: 경로 목록 "blackList": { "kakao.com": ["/talk", "=/pay"] } // → result["kakao.com"] = ["/talk", "=/pay"] // ③ 이중 인코딩 "blackList": "{ \"kakao.com\": [\"/talk\"] }" // → FindString() 감지 → JSONReader::Read() 재파싱 → Dict로 처리 // ④ 쿼리스트링 포함 URL "whiteList": ["https://example.com/path?ref=abc"] // → query 있음 → subdomain_host 세팅 안 됨 // → value = "https://example.com/path?ref=abc" // ⑤ 서브도메인 중복 merge "whiteList": ["https://naver.com/news", "https://naver.com/sports"] // → result["naver.com"] = ["@https://naver.com/news", "@https://naver.com/sports"]
주요 테스트 케이스
테스트명검증 내용
WhiteList_ArrayFormat_DomainsAndPaths Array 포맷 파싱 결과 host 분리 및 @ 패턴 저장 확인
BlackList_DictFormat_PathsPreserved Dict 포맷 경로 목록이 그대로 보존되는지 확인
DoubleEncodedJsonString_WithQueryPattern 이중 인코딩 문자열 재파싱 후 올바른 패턴 생성 확인
Subdomain_Blacklist_Priority_Over_Whitelist 서브도메인 blacklist가 whitelist보다 우선함을 파싱 후 필터로 확인
ArrayFormat_QueryString_CapturedAsPattern 쿼리스트링 포함 URL → subdomain_host 미설정, 쿼리 포함 value로 저장
DictBlacklist_ListWhitelist_SubdomainAllowed Dict blacklist + List whitelist 혼합 포맷에서 서브도메인 허용 확인
BlacklistOnly_ListFormat_RootUrl 루트 URL을 List blacklist에 등록 → subdomain_host로 서브도메인도 차단

5. ObigoJsonConfigTest — obigo.com JSON 파싱 검증

obigo.com 실서비스에 사용하는 JSON 설정을 ParseDomainPatternMap으로 파싱한 뒤 필터 동작을 검증합니다. List/Dict/Unsupported 혼합 포맷과 서브도메인 동작이 핵심입니다.
constexpr char kJson[] = R"({ "whiteList": ["https://obigo.com"], // List 포맷 → root URL → subdomain_host="obigo.com" "blackList": ["https://obigo.com/about-us"], // List 포맷 → kUrlPrefix "unSupportedDomainList": { // Dict 포맷 → kBareSubstring "https://obigo.com": ["promotion", "sdk"], "thread.com": ["oauth"] } })"; filter_.Init("https://www.obigo.com", white, black, unsupported);

6. ObigoUrlFilterTest — obigo.com C++ Map 직접 구성

ObigoJsonConfigTest와 동일한 설정을 JSON 파싱 없이 C++ Map으로 직접 구성합니다. 두 클래스의 결과가 동일해야 ParseDomainPatternMap의 파싱 정합성이 보장됩니다.
DomainPatternMap white = {{"obigo.com", {"@https://obigo.com/"}}}; DomainPatternMap black = {{"obigo.com", {"@https://obigo.com/about-us"}}}; DomainPatternMap unsupported = { {"obigo.com", {"promotion", "sdk"}}, {"thread.com", {"oauth"}}, }; filter_.Init("https://www.obigo.com", white, black, unsupported); // ↑ ObigoJsonConfigTest와 동일 결과 기대
테스트 케이스 및 결과는 ObigoJsonConfigTest와 동일합니다. 이 클래스는 JSON 파싱 레이어가 개입하지 않으므로 파싱 버그를 격리할 수 있습니다.

7. MkAudioCarTest — 매일경제 오디오카

매일경제 오디오카 서비스에 적용된 설정입니다. host_url_로 단일 진입점을 보호하고, www.mk.co.kr 루트를 blacklist로 전체 차단한 뒤 필요한 도메인만 whitelist에 등록합니다.
filter_.Init("https://www.mk.co.kr/audio-car", white, black); DomainPatternMap white = { {"www.mk.co.kr", {"@https://www.mk.co.kr/audio-car"}}, {"m.mk.co.kr", {"@https://m.mk.co.kr/audio-car"}}, {"account.mk.co.kr", {"@https://account.mk.co.kr/p/login/", "@https://account.mk.co.kr/p"}}, }; DomainPatternMap black = { {"www.mk.co.kr", {"@https://www.mk.co.kr/premier", "@https://www.mk.co.kr/"}}, // 루트 → www 전체 차단 {"account.mk.co.kr", {"@https://account.mk.co.kr/p/join/", "@https://account.mk.co.kr/m/join/"}}, };
